# TrashDrop Voice Navigation — Recording Script

> **Purpose**: Script for native speakers to record ~35 navigation phrases per language.
> Each language needs **one male** and **one female** voice recording.
>
> **Placeholders**: `{destination}`, `{steps}`, `{distance}` appear in some phrases.
> For recording, use these example substitutions:
> - `{destination}` → **"Madina Market"**
> - `{steps}` → **"5"**
> - `{distance}` → **"200 meters"** / local equivalent
>
> **Recording notes**:
> - Speak clearly at a **natural, moderate pace**
> - Leave **0.5s silence** before and after each phrase
> - Record each phrase as a **separate audio file**
> - File naming: `{lang}_{gender}_{phrase_key}.wav` (e.g. `tw_male_turn_left.wav`)

## Recording Checklist

### Per Language (repeat for each of 7 languages)

- [ ] **Male speaker** — Record all 35 phrases
- [ ] **Female speaker** — Record all 35 phrases
- [ ] Review recordings for clarity and correct pronunciation
- [ ] Trim silence to 0.5s lead-in / lead-out
- [ ] Export as WAV (44.1kHz, 16-bit mono)
- [ ] Name files: `{lang}_{gender}_{phrase_key}.wav`

```
tw_male_turn_left.wav
tw_female_turn_left.wav

dag_male_roundabout.wav

```

### Phrases with Dynamic Variables

The following phrases contain `{variable}` placeholders. Record them with the example values shown in the tables above. At runtime, the app will stitch audio clips or overlay TTS for the variable portions.

| Key | Variables | Example Substitution |
|-----|-----------|---------------------|
| starting_navigation | `{destination}` | "Madina Market" |
| route_calculated | `{steps}` | "5" |
| in_distance | `{distance}`, `{instruction}` | "200 meters", "Turn left" |
| you_have_arrived | `{destination}` | "Madina Market" |
| distance_remaining | `{distance}` | "200 meters" |
| within_geofence | `{distance}` | "200 meters" |

> **Note**: For Phase 2 integration, phrases with variables will be split into segments
> (e.g. "Starting navigation to" + `[dynamic destination name]`) so that only the
> static portions need pre-recorded audio. The dynamic portions will use TTS fallback
> until Phase 3 (WAXAL TTS models) is ready.
